044/**
045 * <p>Parses command line arguments, using a syntax that attempts to take from the best of POSIX {@code getopt()}
046 * and GNU {@code getopt_long()}.</p>
047 *
048 * <p>This parser supports short options and long options.</p>
049 *
050 * <ul>
051 *   <li><dfn>Short options</dfn> begin with a single hyphen ("<kbd>-</kbd>") followed by a single letter or digit,
052 *   or question mark ("<kbd>?</kbd>"), or dot ("<kbd>.</kbd>").</li>
053 *
054 *   <li>Short options can accept single arguments. The argument can be made required or optional. The option's
055 *   argument can occur:
056 *     <ul>
057 *       <li>in the slot after the option, as in <kbd>-d /tmp</kbd></li>
058 *       <li>right up against the option, as in <kbd>-d/tmp</kbd></li>
059 *       <li>right up against the option separated by an equals sign (<kbd>"="</kbd>), as in <kbd>-d=/tmp</kbd></li>
060 *     </ul>
061 *   To specify <em>n</em> arguments for an option, specify the option <em>n</em> times, once for each argument,
062 *   as in <kbd>-d /tmp -d /var -d /opt</kbd>; or, when using the
063 *   {@linkplain ArgumentAcceptingOptionSpec#withValuesSeparatedBy(char) "separated values"} clause of the "fluent
064 *   interface" (see below), give multiple values separated by a given character as a single argument to the
065 *   option.</li>
066 *
067 *   <li>Short options can be clustered, so that <kbd>-abc</kbd> is treated as <kbd>-a -b -c</kbd>. If a short option
068 *   in the cluster can accept an argument, the remaining characters are interpreted as the argument for that
069 *   option.</li>
070 *
071 *   <li>An argument consisting only of two hyphens (<kbd>"--"</kbd>) signals that the remaining arguments are to be
072 *   treated as non-options.</li>
073 *
074 *   <li>An argument consisting only of a single hyphen is considered a non-option argument (though it can be an
075 *   argument of an option). Many Unix programs treat single hyphens as stand-ins for the standard input or standard
076 *   output streams.</li>
077 *
078 *   <li><dfn>Long options</dfn> begin with two hyphens (<kbd>"--"</kbd>), followed by multiple letters, digits,
079 *   hyphens, question marks, or dots. A hyphen cannot be the first character of a long option specification when
080 *   configuring the parser.</li>
081 *
082 *   <li>You can abbreviate long options, so long as the abbreviation is unique.</li>
083 *
084 *   <li>Long options can accept single arguments.  The argument can be made required or optional.  The option's
085 *   argument can occur:
086 *     <ul>
087 *       <li>in the slot after the option, as in <kbd>--directory /tmp</kbd></li>
088 *       <li>right up against the option separated by an equals sign (<kbd>"="</kbd>), as in
089 *       <kbd>--directory=/tmp</kbd>
090 *     </ul>
091 *   Specify multiple arguments for a long option in the same manner as for short options (see above).</li>
092 *
093 *   <li>You can use a single hyphen (<kbd>"-"</kbd>) instead of a double hyphen (<kbd>"--"</kbd>) for a long
094 *   option.</li>
095 *
096 *   <li>The option <kbd>-W</kbd> is reserved.  If you tell the parser to {@linkplain
097 *   #recognizeAlternativeLongOptions(boolean) recognize alternative long options}, then it will treat, for example,
098 *   <kbd>-W foo=bar</kbd> as the long option <kbd>foo</kbd> with argument <kbd>bar</kbd>, as though you had written
099 *   <kbd>--foo=bar</kbd>.</li>
100 *
101 *   <li>You can specify <kbd>-W</kbd> as a valid short option, or use it as an abbreviation for a long option, but
102 *   {@linkplain #recognizeAlternativeLongOptions(boolean) recognizing alternative long options} will always supersede
103 *   this behavior.</li>
104 *
105 *   <li>You can specify a given short or long option multiple times on a single command line. The parser collects
106 *   any arguments specified for those options as a list.</li>
107 *
108 *   <li>If the parser detects an option whose argument is optional, and the next argument "looks like" an option,
109 *   that argument is not treated as the argument to the option, but as a potentially valid option. If, on the other
110 *   hand, the optional argument is typed as a derivative of {@link Number}, then that argument is treated as the
111 *   negative number argument of the option, even if the parser recognizes the corresponding numeric option.
112 *   For example:
113 *   <pre><code>
114 *     OptionParser parser = new OptionParser();
115 *     parser.accepts( "a" ).withOptionalArg().ofType( Integer.class );
116 *     parser.accepts( "2" );
117 *     OptionSet options = parser.parse( "-a", "-2" );
118 *   </code></pre>
119 *   In this case, the option set contains <kbd>"a"</kbd> with argument <kbd>-2</kbd>, not both <kbd>"a"</kbd> and
120 *   <kbd>"2"</kbd>. Swapping the elements in the <em>args</em> array gives the latter.</li>
121 * </ul>
122 *
123 * <p>There are two ways to tell the parser what options to recognize:</p>
124 *
125 * <ol>
126 *   <li>A "fluent interface"-style API for specifying options, available since version 2. Sentences in this fluent
127 *   interface language begin with a call to {@link #accepts(String) accepts} or {@link #acceptsAll(Collection)
128 *   acceptsAll} methods; calls on the ensuing chain of objects describe whether the options can take an argument,
129 *   whether the argument is required or optional, to what type arguments of the options should be converted if any,
130 *   etc. Since version 3, these calls return an instance of {@link OptionSpec}, which can subsequently be used to
131 *   retrieve the arguments of the associated option in a type-safe manner.</li>
132 *
133 *   <li>Since version 1, a more concise way of specifying short options has been to use the special {@linkplain
134 *   #OptionParser(String) constructor}. Arguments of options specified in this manner will be of type {@link String}.
135 *   Here are the rules for the format of the specification strings this constructor accepts:
136 *
137 *     <ul>
138 *       <li>Any letter or digit is treated as an option character.</li>
139 *
140 *       <li>If an option character is followed by a single colon (<kbd>":"</kbd>), then the option requires an
141 *       argument.</li>
142 *
143 *       <li>If an option character is followed by two colons (<kbd>"::"</kbd>), then the option accepts an optional
144 *       argument.</li>
145 *
146 *       <li>Otherwise, the option character accepts no argument.</li>
147 *
148 *       <li>If the option specification string begins with a plus sign (<kbd>"+"</kbd>), the parser will behave
149 *       "POSIX-ly correct".</li>
150 *
151 *       <li>If the option specification string contains the sequence <kbd>"W;"</kbd> (capital W followed by a
152 *       semicolon), the parser will recognize the alternative form of long options.</li>
153 *     </ul>
154 *   </li>
155 * </ol>
156 *
157 * <p>Each of the options in a list of options given to {@link #acceptsAll(Collection) acceptsAll} is treated as a
158 * synonym of the others.  For example:
159 *   <pre>
160 *     <code>
161 *     OptionParser parser = new OptionParser();
162 *     parser.acceptsAll( asList( "w", "interactive", "confirmation" ) );
163 *     OptionSet options = parser.parse( "-w" );
164 *     </code>
165 *   </pre>
166 * In this case, <code>options.{@link OptionSet#has(String) has}</code> would answer {@code true} when given arguments
167 * <kbd>"w"</kbd>, <kbd>"interactive"</kbd>, and <kbd>"confirmation"</kbd>. The {@link OptionSet} would give the same
168 * responses to these arguments for its other methods as well.</p>
169 *
170 * <p>By default, as with GNU {@code getopt()}, the parser allows intermixing of options and non-options. If, however,
171 * the parser has been created to be "POSIX-ly correct", then the first argument that does not look lexically like an
172 * option, and is not a required argument of a preceding option, signals the end of options. You can still bind
173 * optional arguments to their options using the abutting (for short options) or <kbd>=</kbd> syntax.</p>
174 *
175 * <p>Unlike GNU {@code getopt()}, this parser does not honor the environment variable {@code POSIXLY_CORRECT}.
176 * "POSIX-ly correct" parsers are configured by either:</p>
177 *
178 * <ol>
179 *   <li>using the method {@link #posixlyCorrect(boolean)}, or</li>
180 *
181 *   <li>using the {@linkplain #OptionParser(String) constructor} with an argument whose first character is a plus sign
182 *   (<kbd>"+"</kbd>)</li>
183 * </ol>
184 *
185 * @author <a href="mailto:pholser@alumni.rice.edu">Paul Holser</a>
186 * @see <a href="http://www.gnu.org/software/libc/manual">The GNU C Library</a>
187 */
188public class OptionParser {
189    private final AbbreviationMap<AbstractOptionSpec<?>> recognizedOptions;
190    private OptionParserState state;
191    private boolean posixlyCorrect;
192    private HelpFormatter helpFormatter = new BuiltinHelpFormatter();
193
194    /**
195     * Creates an option parser that initially recognizes no options, and does not exhibit "POSIX-ly correct"
196     * behavior.
197     */
198    public OptionParser() {
199        recognizedOptions = new AbbreviationMap<AbstractOptionSpec<?>>();
200        state = moreOptions( false );
201    }
202
203    /**
204     * Creates an option parser and configures it to recognize the short options specified in the given string.
205     *
206     * Arguments of options specified this way will be of type {@link String}.
207     *
208     * @param optionSpecification an option specification
209     * @throws NullPointerException if {@code optionSpecification} is {@code null}
210     * @throws OptionException if the option specification contains illegal characters or otherwise cannot be
211     * recognized
212     */
213    public OptionParser( String optionSpecification ) {
214        this();
215
216        new OptionSpecTokenizer( optionSpecification ).configure( this );
217    }
218
219    /**
220     * <p>Tells the parser to recognize the given option.</p>
221     *
222     * <p>This method returns an instance of {@link OptionSpecBuilder} to allow the formation of parser directives
223     * as sentences in a fluent interface language. For example:</p>
224     *
225     * <pre><code>
226     *   OptionParser parser = new OptionParser();
227     *   parser.<strong>accepts( "c" )</strong>.withRequiredArg().ofType( Integer.class );
228     * </code></pre>
229     *
230     * <p>If no methods are invoked on the returned {@link OptionSpecBuilder}, then the parser treats the option as
231     * accepting no argument.</p>
232     *
233     * @param option the option to recognize
234     * @return an object that can be used to flesh out more detail about the option
235     * @throws OptionException if the option contains illegal characters
236     * @throws NullPointerException if the option is {@code null}
237     */
238    public OptionSpecBuilder accepts( String option ) {
239        return acceptsAll( singletonList( option ) );
240    }
241
242    /**
243     * Tells the parser to recognize the given option.
244     *
245     * @see #accepts(String)
246     * @param option the option to recognize
247     * @param description a string that describes the purpose of the option.  This is used when generating help
248     * information about the parser.
249     * @return an object that can be used to flesh out more detail about the option
250     * @throws OptionException if the option contains illegal characters
251     * @throws NullPointerException if the option is {@code null}
252     */
253    public OptionSpecBuilder accepts( String option, String description ) {
254        return acceptsAll( singletonList( option ), description );
255    }
256
257    /**
258     * Tells the parser to recognize the given options, and treat them as synonymous.
259     *
260     * @see #accepts(String)
261     * @param options the options to recognize and treat as synonymous
262     * @return an object that can be used to flesh out more detail about the options
263     * @throws OptionException if any of the options contain illegal characters
264     * @throws NullPointerException if the option list or any of its elements are {@code null}
265     */
266    public OptionSpecBuilder acceptsAll( Collection<String> options ) {
267        return acceptsAll( options, "" );
268    }
269
270    /**
271     * Tells the parser to recognize the given options, and treat them as synonymous.
272     *
273     * @see #acceptsAll(Collection)
274     * @param options the options to recognize and treat as synonymous
275     * @param description a string that describes the purpose of the option.  This is used when generating help
276     * information about the parser.
277     * @return an object that can be used to flesh out more detail about the options
278     * @throws OptionException if any of the options contain illegal characters
279     * @throws NullPointerException if the option list or any of its elements are {@code null}
280     * @throws IllegalArgumentException if the option list is empty
281     */
282    public OptionSpecBuilder acceptsAll( Collection<String> options, String description ) {
283        if ( options.isEmpty() )
284            throw new IllegalArgumentException( "need at least one option" );
285
286        ensureLegalOptions( options );
287
288        return new OptionSpecBuilder( this, options, description );
289    }
290
291    /**
292     * Tells the parser whether or not to behave "POSIX-ly correct"-ly.
293     *
294     * @param setting {@code true} if the parser should behave "POSIX-ly correct"-ly
295     */
296    public void posixlyCorrect( boolean setting ) {
297        posixlyCorrect = setting;
298        state = moreOptions( setting );
299    }
300
301    boolean posixlyCorrect() {
302        return posixlyCorrect;
303    }
304
305    /**
306     * Tells the parser either to recognize or ignore <kbd>"-W"</kbd>-style long options.
307     *
308     * @param recognize {@code true} if the parser is to recognize the special style of long options
309     */
310    public void recognizeAlternativeLongOptions( boolean recognize ) {
311        if ( recognize )
312            recognize( new AlternativeLongOptionSpec() );
313        else
314            recognizedOptions.remove( String.valueOf( RESERVED_FOR_EXTENSIONS ) );
315    }
316
317    void recognize( AbstractOptionSpec<?> spec ) {
318        recognizedOptions.putAll( spec.options(), spec );
319    }
320
321    /**
322     * Writes information about the options this parser recognizes to the given output sink.
323     *
324     * The output sink is flushed, but not closed.
325     *
326     * @param sink the sink to write information to
327     * @throws IOException if there is a problem writing to the sink
328     * @throws NullPointerException if {@code sink} is {@code null}
329     * @see #printHelpOn(Writer)
330     */
331    public void printHelpOn( OutputStream sink ) throws IOException {
332        printHelpOn( new OutputStreamWriter( sink ) );
333    }
334
335    /**
336     * Writes information about the options this parser recognizes to the given output sink.
337     *
338     * The output sink is flushed, but not closed.
339     *
340     * @param sink the sink to write information to
341     * @throws IOException if there is a problem writing to the sink
342     * @throws NullPointerException if {@code sink} is {@code null}
343     * @see #printHelpOn(OutputStream)
344     */
345    public void printHelpOn( Writer sink ) throws IOException {
346        sink.write( helpFormatter.format( recognizedOptions.toJavaUtilMap() ) );
347        sink.flush();
348    }
349
350    /**
351     * Tells the parser to use the given formatter when asked to {@linkplain #printHelpOn(java.io.Writer)} print help}.
352     *
353     * @param formatter the formatter to use for printing help
354     * @throws NullPointerException if the formatter is {@code null}
355     */
356    public void formatHelpWith( HelpFormatter formatter ) {
357        if ( formatter == null )
358            throw new NullPointerException();
359
360        helpFormatter = formatter;
361    }
362
363    /**
364     * Parses the given command line arguments according to the option specifications given to the parser.
365     *
366     * @param arguments arguments to parse
367     * @return an {@link OptionSet} describing the parsed options, their arguments, and any non-option arguments found
368     * @throws OptionException if problems are detected while parsing
369     * @throws NullPointerException if the argument list is {@code null}
370     */
371    public OptionSet parse( String... arguments ) {
372        ArgumentList argumentList = new ArgumentList( arguments );
373        OptionSet detected = new OptionSet( defaultValues() );
374
375        while ( argumentList.hasMore() )
376            state.handleArgument( this, argumentList, detected );
377
378        reset();
379        
380        ensureRequiredOptions( detected );
381        
382        return detected;
383    }
384    
385    private void ensureRequiredOptions( OptionSet options ) {
386        Collection<String> missingRequiredOptions = new HashSet<String>();
387        for ( AbstractOptionSpec<?> each : recognizedOptions.toJavaUtilMap().values() ) {
388            if ( each.isRequired() && !options.has( each ) )
389                missingRequiredOptions.addAll( each.options() );
390        }
391
392        if ( !missingRequiredOptions.isEmpty() )
393            throw new MissingRequiredOptionException( missingRequiredOptions );
394    }
395
396    void handleLongOptionToken( String candidate, ArgumentList arguments, OptionSet detected ) {
397        KeyValuePair optionAndArgument = parseLongOptionWithArgument( candidate );
398
399        if ( !isRecognized( optionAndArgument.key ) )
400            throw unrecognizedOption( optionAndArgument.key );
401
402        AbstractOptionSpec<?> optionSpec = specFor( optionAndArgument.key );
403        optionSpec.handleOption( this, arguments, detected, optionAndArgument.value );
404    }
405
406    void handleShortOptionToken( String candidate, ArgumentList arguments, OptionSet detected ) {
407        KeyValuePair optionAndArgument = parseShortOptionWithArgument( candidate );
408
409        if ( isRecognized( optionAndArgument.key ) ) {
410            specFor( optionAndArgument.key ).handleOption( this, arguments, detected, optionAndArgument.value );
411        }
412        else
413            handleShortOptionCluster( candidate, arguments, detected );
414    }
415
416    private void handleShortOptionCluster( String candidate, ArgumentList arguments, OptionSet detected ) {
417        char[] options = extractShortOptionsFrom( candidate );
418        validateOptionCharacters( options );
419
420        for ( int i = 0; i < options.length; i++ ) {
421            AbstractOptionSpec<?> optionSpec = specFor( options[ i ] );
422
423            if ( optionSpec.acceptsArguments() && options.length > i + 1 ) {
424                String detectedArgument = String.valueOf( options, i + 1, options.length - 1 - i );
425                optionSpec.handleOption( this, arguments, detected, detectedArgument );
426                break;
427            }
428
429            optionSpec.handleOption( this, arguments, detected, null );
430        }
431    }
432
433    void noMoreOptions() {
434        state = OptionParserState.noMoreOptions();
435    }
436
437    boolean looksLikeAnOption( String argument ) {
438        return isShortOptionToken( argument ) || isLongOptionToken( argument );
439    }
440
441    private boolean isRecognized( String option ) {
442        return recognizedOptions.contains( option );
443    }
444
445    private AbstractOptionSpec<?> specFor( char option ) {
446        return specFor( String.valueOf( option ) );
447    }
448
449    private AbstractOptionSpec<?> specFor( String option ) {
450        return recognizedOptions.get( option );
451    }
452
453    private void reset() {
454        state = moreOptions( posixlyCorrect );
455    }
456
457    private static char[] extractShortOptionsFrom( String argument ) {
458        char[] options = new char[ argument.length() - 1 ];
459        argument.getChars( 1, argument.length(), options, 0 );
460
461        return options;
462    }
463
464    private void validateOptionCharacters( char[] options ) {
465        for ( char each : options ) {
466            String option = String.valueOf( each );
467
468            if ( !isRecognized( option ) )
469                throw unrecognizedOption( option );
470
471            if ( specFor( option ).acceptsArguments() )
472                return;
473        }
474    }
475
476    private static KeyValuePair parseLongOptionWithArgument( String argument ) {
477        return KeyValuePair.valueOf( argument.substring( 2 ) );
478    }
479
480    private static KeyValuePair parseShortOptionWithArgument( String argument ) {
481        return KeyValuePair.valueOf( argument.substring( 1 ) );
482    }
483
484    private Map<String, List<?>> defaultValues() {
485        Map<String, List<?>> defaults = new HashMap<String, List<?>>();
486        for ( Map.Entry<String, AbstractOptionSpec<?>> each : recognizedOptions.toJavaUtilMap().entrySet() )
487            defaults.put( each.getKey(), each.getValue().defaultValues() );
488        return defaults;
489    }
